# 10 测试验收设计

源头文档：`项目介绍.txt`

溯源段落：
- 前端交互层：智能对话、证据展示、ReAct 轨迹、任务清单、人工确认、设备/告警概览、图片上传研判。
- Human-in-the-loop 风险控制：高风险动作暂停当前执行链路，写入 checkpoint 和 review request，同时在终端输出待确认信息；审批通过后通过 LangGraph interrupt / resume 从 checkpoint 继续执行原流程。
- 前端闭环：形成完整“问答 - 研判 - 审核 - 追溯”闭环。

覆盖任务：
- T03 制定测试与验收标准
- T90 设计 AuditEvent 类型
- T94 编写后端单元测试
- T95 编写前端组件测试
- T96 编写端到端验收用例

## 2. 必测业务场景

### 2.1 设备离线排障

输入：

```text
北门摄像头离线了，帮我排查。
```

期望：

- intent = `device_troubleshoot`
- target_agent = `ops-troubleshooter`
- 调用 `query_device_status`
- 调用 `search_security_knowledge`
- 返回排查任务清单
- 返回 evidence 和 trace

### 2.2 告警误报分析

输入：

```text
昨晚周界入侵告警是不是误报？
```

期望：

- intent = `alarm_analysis`
- target_agent = `alarm-analyst`
- 调用 `query_alarm_events`
- 返回误报判断依据
- 返回处置建议

### 2.3 知识库问答

输入：

```text
摄像头接入平台失败有哪些常见原因？
```

期望：

- target_agent = `security-researcher`
- 调用 `search_security_knowledge`
- 回答中包含来源引用
- evidence 不为空

### 2.4 图片物品研判

输入：

```text
上传现场图片，判断是否有风险物品。
```

期望：

- target_agent = `object-analyst`
- 调用 `analyze_security_object`
- 输出场景概述、识别对象、风险等级、处置建议

## 3. 高风险中断恢复验收

### 3.1 审批通过

输入：

```text
重启北门流媒体服务。
```

期望：

```text
RiskGate high
→ Agent interrupt
→ Redis checkpoint 写入
→ PostgreSQL review_request pending
→ 终端输出 [REVIEW_REQUIRED]
→ ChatResponse.interrupted=true
→ approve
→ resume
→ 返回继续后的处理结果
```

必须验证：

- 高风险动作在审批前没有执行。
- 终端提示包含 review_id、risk_level、reason、proposed_action、resume_api。
- resume 不是重新执行整轮 chat。
- resume_token 只能消费一次。
- audit 中有 `agent_interrupted` 和 `agent_resumed`。

### 3.2 审批拒绝

输入：

```text
删除这台设备的配置。
```

期望：

```text
RiskGate high
→ Agent interrupt
→ reject
→ resume
→ Agent 输出终止说明或替代建议
```

必须验证：

- 原高风险动作没有执行。
- review 状态变为 rejected / resumed / cancelled。
- 用户能看到替代建议。

## 4. 多模型 Sub-agent 验收

必须验证：

- `security-researcher` 使用自己的 `model_profile`。
- `ops-troubleshooter` 使用自己的 `model_profile`。
- `object-analyst` 可使用 Vision profile。
- 配置不存在的 model_profile 时服务启动失败。
- `internal_only` 数据不会发送到 `sanitized_only` 或公开 API profile。
- 私有模型失败时不会自动把敏感上下文 fallback 到公开 API。

---

## 5. 审计验收

每个 thread 至少可追溯：

- 用户消息。
- intent route。
- context build 摘要。
- handoff。
- tool call。
- risk decision。
- review decision。
- checkpoint resume。
- assistant response。

审计禁止包含：

- API key。
- 数据库连接串。
- 完整图片 base64。
- 未脱敏设备密码。

## 6. 最小交付标准

首个版本通过以下端到端用例即可进入下一阶段：

1. 设备离线排障闭环。
2. 告警误报分析闭环。
3. 知识库问答闭环。
4. 高风险 interrupt / approve / resume 闭环。
5. 高风险 interrupt / reject / 替代建议闭环。
6. Sub-agent 多模型配置校验。
7. 前端展示 answer、evidence、trace、tasks、review 状态。
