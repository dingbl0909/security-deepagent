# 03 数据存储设计

源头文档：`项目介绍.txt`

溯源段落：
- API 服务层支持会话持久化、审核流转和业务数据查询。
- Harness 包含会话持久化、审计日志、检查点恢复。
- Human-in-the-loop 写入 checkpoint 和 review request。

覆盖任务：
- T20 PostgreSQL schema
- T21 ThreadStore / MessageStore
- T22 ReviewStore
- T23 TaskStore
- T24 DeviceStore / AlarmStore
- T25 Redis SessionStore 和 Checkpoint
- T26 Milvus collection 初始化

---

## 1. 存储分工

| 存储 | 作用 |
|------|------|
| PostgreSQL | 会话、消息、任务、审核、审计、设备/告警镜像 |
| Redis | 短期记忆、LangGraph checkpoint |
| Milvus | 知识库向量、长期经验记忆 |
| MinIO/S3 | 图片和附件 |

---

## 2. PostgreSQL 表

首版表：

- `threads`
- `messages`
- `tasks`
- `task_events`
- `review_requests`
- `handoffs`
- `tool_calls`
- `audit_events`
- `devices`
- `alarms`

`review_requests` 必须包含：

- `id`
- `thread_id`
- `status`
- `risk_level`
- `reason`
- `proposed_action`
- `checkpoint_ref`
- `resume_token_hash`
- `operator_id`
- `decided_at`
- `resumed_at`

---

## 3. Redis Key

```text
agent:session:{thread_id}
agent:checkpoint:{thread_id}
```

`agent:session` 保存摘要、关键事实、当前 agent、active review。

`agent:checkpoint` 由 LangGraph RedisSaver 管理。

---

## 4. Milvus Collection

| Collection | 作用 |
|------------|------|
| `security_knowledge` | SOP 和知识库 chunk |
| `agent_episodic_memory` | 历史处置经验 |

---

## 5. 验收

- 所有事务数据可按 `thread_id` 追溯。
- Redis checkpoint 可支持 resume。
- Milvus 检索结果包含 source 和 score。
- 审核状态机可恢复、可审计。
