# 00 总体架构设计

源头文档：`项目介绍.txt`

溯源段落：
- 技术架构：项目基于 DeepAgents（LangChain + LangGraph）构建 Agent 运行时，后端采用 FastAPI，前端采用 React + TypeScript + Vite，形成“前端交互层 - API 服务层 - Agent 驾驭层 - 工具/Skills/沙箱层”的分层架构。
- 项目实现：Harness 驾驭层统一上下文治理、工具编排、会话持久化、审计日志、检查点恢复；Skills + Sub-agents 场景化编排；Human-in-the-loop 风险控制。

## 1. 项目定位

Security DeepAgent 是面向安防运维场景的私有化 Agent 平台，核心目标是把设备排障、告警分析、知识库问答、图片研判、人工确认和审计追溯放到同一条可控链路里。

项目不是开放式插件平台，不直接引入公共 Skills 市场；所有 Skills、工具、Sub-agent 和模型配置都必须经过本地声明和审核。

## 4. Sub-agent 与模型路由

不同 Sub-agent 可以使用不同大模型，包括私有化部署模型和公开 API。模型选择不由 prompt 临时决定，而由配置和 Harness 控制。

### 4.1 配置维度

每个 Sub-agent 可配置：

```yaml
agents:
  - name: security-researcher
    model_profile: private-qwen-long-context
    tools:
      - search_security_knowledge

  - name: ops-troubleshooter
    model_profile: private-qwen-secure
    tools:
      - query_device_status
      - create_security_todos
      - request_human_review

  - name: object-analyst
    model_profile: public-vision-api
    tools:
      - analyze_security_object
```

`model_profile` 再映射到具体 provider：

```yaml
model_profiles:
  private-qwen-secure:
    provider: openai-compatible
    base_url: http://vllm.internal/v1
    model: qwen-security-32b
    data_policy: internal_only

  public-vision-api:
    provider: openai-compatible
    base_url: https://api.example.com/v1
    model: vision-model
    data_policy: public_api_allowed
```

### 4.2 安全原则

- 设备状态、告警记录、内部 SOP、审计上下文默认只允许进入私有化模型。
- 公开 API 只能接收脱敏后的输入，或只用于不含内部敏感信息的任务。
- Vision 公共 API 若需要处理现场图片，必须先经过数据分级确认和对象存储访问控制。
- 模型 API Key 不进入 prompt、trace、审计明文。
- Sub-agent 不能自行切换模型，只能使用 Harness 下发的 `model_profile`。

### 4.3 运行路径

```text
IntentRouter 选择 target_agent
  → SubAgentRegistry 读取 target_agent.model_profile
  → LLMGatewayFactory 创建对应模型客户端
  → SubAgentRunner 注入对应 LLM + 工具白名单
  → 返回 AgentRunResult
```

## 5. 首版范围

首版必须实现：

- FastAPI `/chat`、`/threads`、`/reviews`、`/devices`、`/alarms`、`/health`、`/ready`。
- PostgreSQL 会话、消息、任务、审核、审计。
- Redis 短期记忆和 LangGraph checkpoint。
- Milvus 知识库和长期经验记忆接口。
- DeepAgents / LangGraph 主 Agent + Sub-agent 编排。
- 工具白名单。
- 高风险 interrupt、终端提示、人工审批、resume。
- React 工作台基础页面。

首版不做：

- 完整 RBAC / SSO。
- 任意 Shell 执行。
- 公共 Skills 市场接入。
- 未脱敏数据发送到公开模型 API。

## 6. 关键验收

- 用户问设备离线问题，系统能返回排查结论、证据、轨迹和任务。
- 用户问告警误报问题，系统能返回告警分析和处置建议。
- 用户上传现场图片，系统能返回场景、对象、风险和建议。
- 用户要求重启、删除、修改配置、升级、下发规则时，系统必须中断流程并等待人工确认。
- 审批通过后必须从 checkpoint 继续；审批拒绝后必须恢复流程并给出终止说明或替代建议。
- 每个 Sub-agent 使用的模型 profile 可审计、可配置、可测试。
